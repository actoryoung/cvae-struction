# MOSI 实验结果问题分析

## 🔴 Bug 发现：NIG 损失符号错误（已修复）

**根因**：Amini et al. (NeurIPS 2020) Eq. (9) 的 NIG NLL 公式为：
```
L = ½·log(π/ν) - α·log(Ω) + (α+½)·log((y-γ)²ν + Ω) + log(Γ(α)/Γ(α+½))
```

实现中误写为 `+ α·log(Ω)`（应为 `- α·log(Ω)`）。

**影响**：
- 正确公式：当 β → 0，Ω → 0，log(Ω) → -∞，`-α·log(Ω)` → **+∞**（惩罚小 β）
- 错误公式：当 β → 0，`+α·log(Ω)` → **-∞**（奖励小 β）
- → 模型学到让 β → 0，驱动 loss → -∞，V2 崩溃

**修复**：`evidential_uadg.py` 和 `evidential_v2.py` 中均已修正。修复后 NIG loss 稳定在 1-3 范围。

## 问题记录

### 问题 1：Evidential 在缺失模态下反而更差

| Scenario | concat Acc-2 | evidential Acc-2 | Δ |
|----------|-------------|-----------------|---|
| Full | 0.796 | 0.804 | +0.8 ✅ |
| Missing Text | 0.599 | 0.599 | 0.0 |
| Missing Audio | 0.795 | 0.801 | +0.6 |
| Missing Vision | 0.799 | 0.798 | **-0.1** ❌ |

**预期**：evidential 应在缺失模态时明显优于 concat
**实际**：缺失视觉时反而更差，缺失文本时完全无差别

### 问题 2：文本主导效应

缺失文本时所有模型 Acc-2 从 ~0.80 暴跌到 ~0.60。MOSI 中文本是绝对主导模态，缺失后其他模态几乎无法补偿。

### 问题 3：Evidential 收敛慢

Evidential best epoch = 18，concat best epoch = 12。NIG 损失优化更难。

## 根因分析

### 根因 1：架构设计有问题

当前设计：**每个模态独立预测 γ_m 和证据 ν_m** → 然后加权融合

```
问题：单个模态（纯音频/纯视觉）预测情感的能力很弱
     → 各模态的证据 ν_m 都差不多（都不太可信）
     → 门控退化为近似均匀权重
     → 与 concat 无本质区别
```

### 根因 2：证据定义不匹配任务

ν_m 定义为"该模态独立预测情感的证据量"，但在 MSA 中，**单模态预测能力弱 ≠ 该模态对融合无贡献**。音频可能单独预测不准，但能提供文本无法捕捉的情绪线索。

### 根因 3：缺失模态时证据归零太粗暴

直接设 ν=0 让梯度完全切断，模型无法学习"如何在缺失模态下调整其他模态的证据"

## 解决方案

### 方案 A（推荐）：改为"融合后单 NIG 头"架构

```
Text  ──► Encoder ──► h_t ──┐
Audio ──► Encoder ──► h_a ──┤
Vision──► Encoder ──► h_v ──┤
                              │
                    ┌────────▼────────┐
                    │ Evidence Heads   │  ← 每模态输出 evidence (scalar)
                    │ e_t, e_a, e_v   │     = softplus(MLP(h_m))
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ Weighted Fusion  │  ← h_fused = Σ(e_m/Σe_j)·h_m
                    └────────┬────────┘
                             │
                    ┌────────▼────────┐
                    │ Single NIG Head  │  ← 单个 NIG 头 → (γ, ν, α, β)
                    │ on fused result  │
                    └────────┬────────┘
                             │
                          output
```

- 证据 e_m 含义变为"该模态表示的信息量/质量"（而非独立预测能力）
- 融合后单个 NIG 头做最终预测
- 更符合直觉：**证据用于 "听谁的"，NIG 用于 "有多确定"**

### 方案 B：证据对比学习

在 modality dropout 时，用对比损失强制：
- 缺失某模态时的 fused prediction ≈ 完整模态时的 fused prediction
- 同时 evidence 头学习区分"有用 vs 无用"的模态

### 方案 C：梯度友好的缺失处理

不直接设 ν=0，而是：
- 训练时：对缺失模态，用 Gumbel-Softmax 做软 mask
- 推理时：硬 mask
- 让梯度能流过缺失路径

## 下一步

1. **先实现方案 A**（改动最小，理论最清晰）
2. 对比方案 A 与当前 V2 在 MOSI 上的表现
3. 根据结果决定是否加入方案 B/C
